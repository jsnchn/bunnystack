### 2. Local Postgres + DB Migrations

Get the database layer live so auth and schema actually work out of the box.

**Tasks:**
- Add `docker-compose.yml` at root with a Postgres 17 service
- Create `.env.example` with `DATABASE_URL` and `PORT` defaults
- Generate initial Drizzle migration from `schema.ts`
- Add `db:up`, `db:migrate`, `db:generate` scripts in root `package.json`
- Verify migration runs and schema is applied

**Files:**
- Create: `docker-compose.yml`, `.env.example`
- Modify: `package.json`

**Validation:**
- `docker compose up -d db` starts Postgres
- `bun run --filter @bunnystack/db generate && bun run --filter @bunnystack/db migrate` applies schema
- `drizzle-kit studio` shows the `users` table

---

### 3. Better-Auth Integration with ElysiaJS

The auth package exists but is not wired into the API server. Better-Auth
needs ElysiaJS plugins/routes mounted.

**Tasks:**
- Install `@better-auth/elysia` in `apps/api`
- Mount Better-Auth on `/api/auth` in the Elysia server
- Set up session validation middleware for protected routes
- Expand `schema.ts` with Better-Auth's required tables (sessions, accounts,
  verifications) — Better-Auth can auto-generate these via Drizzle adapter

**Files:**
- Modify: `apps/api/src/index.ts` (mount auth),
          `apps/api/package.json` (add `@better-auth/elysia`),
          `packages/auth/src/auth.ts` (add Drizzle adapter config)
          `packages/db/src/schema.ts` (add auth tables)

**Validation:**
- `bun run check-types` passes
- `curl localhost:3000/api/auth/sign-up` returns a response
